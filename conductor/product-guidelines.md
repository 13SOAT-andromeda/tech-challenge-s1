# Product Guidelines

## Prose Style
- **Clear and Professional:** Use direct, professional language in all documentation and API responses.
- **English-First:** Primary documentation and code comments should be in English.
- **Portuguese Localization:** Support for localized messages in the UI (if applicable) and user-facing communications.

## Branding
- **Garage Management:** Consistent use of "Tech Challenge Garage" (or similar) branding in all communications.
- **Clean Interface:** Focus on a clean, functional API design.

## UX Principles
- **Efficiency First:** Fast and responsive API endpoints.
- **Self-Documenting:** Use meaningful names for API fields and endpoints.
- **Comprehensive Errors:** Provide clear, actionable error messages with appropriate HTTP status codes.
- **Interactive Docs:** Keep Swagger/Redoc documentation up-to-date with all changes.
